# Telegram AI Moderation Bot

Telegram-бот на **aiogram 3.x** с AI-модерацией через **Gemini API**.

## Возможности

- **Доступ в ЛС**: владелец + суб-админы; остальным — отказ; спам → бан на 7 дней
- **Управление чатами из ЛС**: правила, интервал, модерация, наказания, статистика
- **Суб-админы**: привязка к своим чатам, индивидуальный дневной лимит запросов
- **AI-модерация**: `none` (не нарушил) / `pardon` / `punish` с логированием
- **Ротация Gemini**: 2 проекта, 5 моделей, очередь 15 RPM
- **Конфиг в файлах**: `config/settings.json` + `config/secrets.json`

## Быстрый старт

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
# Вставьте токен от @BotFather в config/secrets.json → bot_token
python3 -m bot.main
```

> `config/secrets.json` в репозитории — держите репозиторий **приватным**.

## Конфигурация

### config/secrets.json
```json
{
  "bot_token": "YOUR_BOT_TOKEN",
  "gemini_api_keys": ["KEY_PROJECT_1", "KEY_PROJECT_2"],
  "proxy": {
    "enabled": true,
    "type": "socks5",
    "host": "185.13.225.37",
    "port": 49156,
    "username": "login",
    "password": "password"
  }
}
```

Для HTTP-прокси укажите `"type": "http"` и порт `49155`.

### config/settings.json
- `owner_id` — Telegram ID владельца
- `rpm_limit` — запросов в минуту (15)
- `rpd_per_model` — лимит на модель/день (1000)
- `default_batch_interval` — интервал слотов батчинга (30 сек → :00 и :30 UTC)
- `chat_history_limit` — сообщений в БД на чат для контекста ИИ (200)
- `log_clean_checks` — логировать «не нарушил» в консоль

## Команды

| Команда | Где | Описание |
|---|---|---|
| `/admin` | ЛС | Панель управления |
| `/linkchat` | Группа | Привязать чат к аккаунту |
| `/addadmin <id> <limit>` | ЛС (владелец) | Добавить суб-админа |
| `/cleardb yes` | ЛС (владелец) | Очистить БД после тестов (лимиты Gemini, наказания) |
| `/punishments` | Группа | Активные наказания |
| `/setrules`, `/setinterval`, `/mod` | Группа | Настройки |

## Решения AI

| action | Значение |
|---|---|
| `none` | Не нарушил — ничего не делать (логируется) |
| `pardon` | Устное предупреждение |
| `punish` | Мут / наказание |

## Суб-админы

1. Владелец: `/addadmin 123456789 500` или через /admin → Суб-админы
2. Суб-админ добавляет бота в группу → `/linkchat`
3. Управляет только своими чатами из ЛС
4. Расходует свой дневной лимит запросов

## Совместимость с Ирисом (@iris_cm_bot)

**Параллельная работа — да.** Оба бота могут быть админами в одном чате и читать сообщения независимо.

**Прямое взаимодействие (отправлять команды Ирису) — нет.** Ирис обрабатывает команды от пользователей, не от других ботов. Настройка `-Боты` / `+Боты` у Ириса запрещает **приглашать** ботов в чат, а не чтение сообщений между ботами.

Наш бот не требует, чтобы Ирис «взаимодействовал с ботами» — он сам анализирует сообщения через Telegram API.
